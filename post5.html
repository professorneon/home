💻 Kernel Nədir? Əməliyyat Sisteminin Ürəyinə Dərin Baxış

Yazan: ISKOPISIX | Tarix: 07.06.2025 | Qovluq: Əməliyyat Sistemləri / Kernel

🔍 Kernel nədir?

Kernel — əməliyyat sisteminin əsas və ən vacib komponentidir. O, kompüterin aparat təminatı (hardware) ilə proqram təminatı (software) arasında təməl körpü funksiyasını yerinə yetirir. Kernel olmadan əməliyyat sistemi yararsızdır, çünki kernel sistemin bütün resurslarını — prosessor vaxtı, yaddaş, giriş-çıxış cihazları və digər hardware komponentlərini — mərkəzləşdirilmiş şəkildə idarə edir və paylayır. Kernelin əsas vəzifəsi, müxtəlif proqramların eyni zamanda kompüterdə problemsiz və təhlükəsiz işləməsini təmin etməkdir. Bunun üçün kernel aparat səviyyəsində idarəetmə həyata keçirir: o, proseslərin işləmə vaxtını balanslaşdırır (prosesslərin planlaşdırılması/scheduling), yaddaşın düzgün bölüşdürülməsini təmin edir, diskə və digər cihazlara giriş-çıxış əməliyyatlarını idarə edir, həmçinin sistemin təhlükəsizliyini və sabitliyini qoruyur. Texniki baxımdan kernel ən yüksək səlahiyyətlə (privilege level, ring 0) işləyir. Bu o deməkdir ki, kernel proqramları aparatın bütün imkanlarına tam girişə malikdir və sistemin təhlükəsizliyini təmin etmək üçün xüsusi qoruma mexanizmləri tətbiq edir. İstifadəçi səviyyəsində işləyən proqramlar isə kernel vasitəsilə hardware resurslarına çatır, kernel onların əməliyyatlarını sinxronlaşdırır və qarşılıqlı ziddiyyətlərin qarşısını alır. Kernel olmadan, kompüterdə eyni vaxtda bir neçə proqramın effektiv işləməsi, sistem resurslarının düzgün idarə olunması, cihazlarla əlaqənin qurulması və əməliyyat sisteminin əsas funksiyalarının icrası mümkün olmazdı. Ona görə də kernel əməliyyat sisteminin ürəyi və beyni sayılır.

⚙️ Kernelin əsas funksiyaları

  • Prosessor idarəsi:

    Kernel prosessorun vaxtını müxtəlif proseslər və ipliklər (thread) arasında ədalətli və səmərəli şəkildə paylayır. O, proseslərin icrasını planlaşdırır (scheduling), yəni hansı prosesin nə vaxt və nə qədər işləyəcəyinə qərar verir. Kernel bunun üçün müxtəlif algoritmlərdən istifadə edir, məsələn:

    • Round Robin – proseslərə bərabər vaxt verilir.
    • Priority Scheduling – öncəlik dərəcəsinə görə.
    • Multi-level Feedback Queue – prosesləri fərqli növbələrə bölür və onları optimallaşdırır.

    Kernel həmçinin proseslər arasında sürətli keçid (context switch) edir, bu da eyni vaxtda çoxlu işlərin aparılmasını — multitasking funksiyasını təmin edir.

  • Yaddaş idarəsi:

    Kernel kompüterin yaddaşını (RAM və virtual yaddaş) idarə edir. Hər prosesə öz unikal yaddaş sahəsi verilir ki, proseslər bir-birinin yaddaşına müdaxilə etməsin. Kernel yaddaşın bölünməsi və səhifələmə (paging) kimi üsullarla yaddaşdan səmərəli istifadəni təmin edir.

    Əgər RAM dolursa, kernel yaddaşın bir hissəsini diskdə saxlayır (swap) və bu sayədə sistemin dayanmadan işləməsinə imkan verir.

  • Fayl sistemi idarəsi:

    Kernel faylların və qovluqların idarəsini həyata keçirir. O, məlumatların disklərdə necə saxlanması, oxunması və yazılması üçün xüsusi qaydalar və metodlar təqdim edir. Kernel müxtəlif fayl sistemlərini dəstəkləyir, məsələn, NTFS, EXT4, FAT32 və s.

    Faylların yaradılması, silinməsi, dəyişdirilməsi və icazələrin təyin edilməsi kimi əməliyyatlar kernel tərəfindən təhlükəsiz və ardıcıl şəkildə idarə olunur.

  • Cihaz sürücüləri:

    Kernel kompüterdəki bütün aparat cihazları ilə əlaqəni təmin etmək üçün cihaz sürücülərindən istifadə edir. Hər bir aparatın özünəməxsus sürücüsü olur ki, kernel vasitəsilə həmin aparatın funksiyaları idarə olunsun.

    Misal üçün, video kart, şəbəkə adapteri, printer kimi cihazların işi sürücülər vasitəsilə kernelə inteqrasiya olunur.

  • Giriş-çıxış əməliyyatları:

    Kernel klaviatura, siçan, printer və digər periferik cihazlarla məlumat mübadiləsini idarə edir. O, cihazlardan gələn məlumatları proqramlara çatdırır və əksinə, proqramlardan gələn əmrləri cihazlara göndərir.

    Kernel həmçinin bu əməliyyatların sinxronizasiyasını və prioritetləşdirilməsini həyata keçir ki, sistem stabil və sürətli işləsin.

  • Təhlükəsizlik və icazələr:

    Kernel sistemin təhlükəsizliyini təmin edir. O, istifadəçilərin və proqramların sistem resurslarına çıxışını məhdudlaşdırır və qoruyur.

    Kernel müxtəlif istifadəçi səviyyələri, hüquqlar və icazələr sistemi vasitəsilə resursların müdafiəsini həyata keçirir. Həmçinin, proseslər arasında izolyasiya yaradır ki, bir proqram digərinin yaddaşına və resurslarına müdaxilə edə bilməsin.

    Əlavə olaraq, kernel təhlükəsizlik protokolları və audit mexanizmləri tətbiq edir, zərərli proqramların sistemə daxil olmasının qarşısını almağa kömək edir.

💡 Kernel növləri

Əməliyyat sistemlərində kernelin müxtəlif arxitektura və dizayn yanaşmaları mövcuddur. Hər bir kernel növü sistemin performansı, stabilliyi, təhlükəsizliyi və genişlənə bilmə qabiliyyəti baxımından fərqli üstünlüklərə və çatışmazlıqlara malikdir. Aşağıda kernel növlərinin əsas kateqoriyaları və onların detallı izahı verilmişdir:

  • Monolitik Kernel:

    Monolitik kernel bütün əsas əməliyyat sistemi funksiyalarını, o cümlədən proses idarəetməsi, yaddaş idarəsi, cihaz sürücüləri və fayl sistemi idarəsini böyük tək bir proqram daxilində icra edir. Bu kernel növü bütün komponentləri bir yerdə saxlayır və onlar bir-biri ilə çox sıx əlaqədə işləyir.

    Üstünlükləri:

    • Yüksək performans — bütün modullar eyni yaddaş məkanında çalışdığı üçün əlaqə sürətlidir.
    • Geniş aparat dəstəyi — çox sayda cihaz sürücüsü kernel daxilində birbaşa idarə olunur.

    Çatışmazlıqları:

    • Stabilik riskləri — bir moduldakı səhv bütün kernelin çöküşünə səbəb ola bilər.
    • Mürəkkəblik — kernelin böyük və kompleks olması onu inkişaf etdirməyi və saxlamanı çətinləşdirir.

    Misal: Linux kernel, Unix kimi klassik əməliyyat sistemlərinin kernel dizaynı monolitikdir.

  • Microkernel:

    Microkernel dizaynında yalnız ən zəruri və əsas funksiyalar kernel səviyyəsində işləyir — məsələn, proseslərarası kommunikasiya (IPC), minimal yaddaş idarəsi və əsas planlaşdırma mexanizmləri. Digər funksiyalar, o cümlədən cihaz sürücüləri, fayl sistemi və şəbəkə xidmətləri kimi modullar isə kernel xaricində, istifadəçi səviyyəsində ayrıca xidmətlər kimi işləyir.

    Üstünlükləri:

    • Yüksək stabillik və təhlükəsizlik — çünki xidmətlər ayrı-ayrı proseslərdə işləyir və birindəki problem bütün sistemə yayılmır.
    • Daha asan genişlənmə və yenilənmə — kernel minimal olduğundan, əlavə xidmətlərin dəyişdirilməsi kernelə təsir etmədən həyata keçirilə bilər.

    Çatışmazlıqları:

    • Performans itkisi — istifadəçi səviyyəsindəki modullarla kernel arasında çoxlu kontekst dəyişməsi və IPC çağırışları sistemin sürətinə təsir edə bilər.
    • İdarəetmənin mürəkkəbliyi — xidmətlər arasında əlaqə və sinxronizasiyanın yüksək səviyyədə koordinasiyası tələb olunur.

    Misal: Minix, QNX, HURD kimi əməliyyat sistemləri microkernel arxitekturasına əsaslanır.

  • Hybrid Kernel:

    Hybrid kernel yanaşması monolitik və microkernel arxitekturalarının elementlərini birləşdirir. Bu kernel növü əsas funksiyaları kernel səviyyəsində saxlayır, lakin bəzi xidmətləri istifadəçi səviyyəsinə çıxararaq stabillik və performans arasında kompromis yaradır.

    Üstünlükləri:

    • Monolitik kernelin yüksək performansını saxlayır.
    • Microkernelin modularlığını və stabilliyini qismən təmin edir.
    • Geniş aparat və proqram təminatı dəstəyi mövcuddur.

    Çatışmazlıqları:

    • Həm microkernel, həm də monolitik kernelin bəzi problemlərini daşıya bilər.
    • Kernelin həcmi və mürəkkəbliyi nisbətən yüksəkdir.

    Misal: Windows NT kernel, macOS XNU kernel bu arxitekturaya misaldır.

  • Exokernel:

    Exokernel kernel dizaynında sistem resurslarının idarəetməsi minimal səviyyədə olur və əsas məqsəd maksimum çeviklik təmin etməkdir. Bu yanaşmada kernel aparat resurslarının paylaşdırılmasını sadə və az müdaxiləli şəkildə həyata keçirir, yüksək səviyyəli sistem xidmətləri isə istifadəçi səviyyəsində işləyir.

    Üstünlükləri:

    • Resurs idarəetməsində maksimum çeviklik və istifadəçi proqramlarına geniş nəzarət imkanı.
    • Az kernel overhead — yəni kernelin özü az resurs istifadə edir.

    Çatışmazlıqları:

    • Çox mürəkkəb proqram təminatı yazılması tələb olunur, çünki yüksək səviyyəli idarəetmə modulunu proqram təminatı təmin etməlidir.
    • Geniş kommersiya və istehlak sistemlərində geniş yayılmayıb, daha çox araşdırma və akademik məqsədlərlə istifadə olunur.

    Misal: Exokernel dizaynı ilk dəfə MIT tərəfindən araşdırılmış və eksperimental əməliyyat sistemlərində tətbiq olunmuşdur.

🔧 Kernelin iş prinsipi

Kernel kompüterin ən əsas və mərkəzi proqram komponentidir və əməliyyat sisteminin “ürəyi” hesab olunur. Digər proqramlardan fərqli olaraq, kernel yüksək səviyyəli hüquqlarla — ring 0 rejimində işləyir. Bu, ona aparat vasitələrinə (CPU, yaddaş, cihazlar) tam və sərbəst çıxış imkanı verir. Ring 0 rejimi ən aşağı səviyyədir və sistemin kritik əməliyyatları məhz burada icra olunur.

İstifadəçi səviyyəsində çalışan proqramlar (məsələn, brauzer, redaktor, oyunlar) birbaşa aparatla işləyə bilmirlər. Onlar kernel vasitəsilə aparat resurslarına çıxış əldə edir. Bu çıxış kernelin xüsusi interfeysləri — sistem çağırışları (system calls) vasitəsilə baş verir. Proqram sistem çağırışı ilə kernelə müraciət edir, kernel isə əməliyyatın yerinə yetirilməsini təmin edir və nəticəni geri qaytarır.

Kernel həmçinin prosessor vaxtını proseslər və ipliklər arasında bölür (prosessor idarəsi). Bu zaman kernel planlaşdırma algoritmlərindən istifadə edir və çoxlu proseslərin eyni vaxtda işləməsi üçün kontekst dəyişmələrini (context switches) həyata keçirir. Bu, istifadəçi üçün paralel işləmə kimi görünür.

Yaddaşın idarəsi kernelin ən vacib funksiyalarından biridir. Kernel proseslər üçün virtual yaddaş məkanları yaradır və yaddaşın qorunmasını təmin edir. Hər proses öz yaddaş sahəsinə sahibdir və kernel vasitəsilə digər proseslərin yaddaşına müdaxilə etmək mümkün deyil. Kernel səhifələmə (paging) və bölmə (segmentation) kimi texnikalardan istifadə edərək yaddaşın fiziki və virtual hissələrini idarə edir.

Kernel həmçinin sinxronizasiya mexanizmlərini təmin edir ki, bu da bir neçə prosesin eyni vaxtda kritik resurslara (məsələn, fayl və ya cihaz) zidd olmadan çıxışını təmin edir. Mutex, semaphore və digər sinxronizasiya vasitələri kernel tərəfindən idarə olunur.

Giriş-çıxış əməliyyatlarında kernel cihaz sürücüləri vasitəsilə periferik qurğuları idarə edir. Kernel bu əməliyyatların təhlükəsiz, ardıcıl və effektiv aparılmasını təmin edir, həmçinin bu proseslərin prioritetləşdirilməsinə nəzarət edir ki, sistem dayanıqlı işləsin.

Qısaca desək, kernel əməliyyat sisteminin mərkəzi idarəedicisi kimi istifadəçi proqramları ilə aparat arasında körpü rolunu oynayır. Onun iş prinsipi — yüksək səlahiyyətlərlə işləyib, resursları bölüşdürmək, təhlükəsizliyi qorumaq və sistemin sabitliyini təmin etməkdən ibarətdir.

📋 Sadə kernel modulu nümunəsi (Linux)

#include <linux/module.h>
#include <linux/kernel.h>

int init_module(void) {
    printk(KERN_INFO "Kernel modulu yükləndi!\n");
    return 0;
}

void cleanup_module(void) {
    printk(KERN_INFO "Kernel modulu çıxarıldı!\n");
}

MODULE_LICENSE("GPL");

Bu sadə Linux kernel modulu nümunəsi yüklənərkən və çıxarılarkən mesajlar göstərir. Kernel modulları kernelə əlavə funksionallıq əlavə etməyə imkan verir.

⚠️ Kernel ilə əlaqəli təhlükəsizlik məsələləri

Kernel sistemin ən əsas və həssas hissəsidir. Onun zəiflikləri bütün sistemi təhlükə altına sala bilər. Buna görə kernelin kodu ciddi yoxlanılır və təhlükəsizlik patch-ləri tez-tez tətbiq olunur. Zərərli proqramlar (rootkitlər) kernel səviyyəsində işləyərək sistemə gizli nəzarət edə bilər, buna görə kernel modullarının etibarlı mənbələrdən olması çox vacibdir.

🚀 Nəticə

Kernel, əməliyyat sisteminin ürəyi kimi işləyir və kompüterin bütün resurslarını idarə edir. Onun düzgün işləməsi sistemin sabitliyi, təhlükəsizliyi və performansı üçün həyati əhəmiyyət daşıyır. Kernel arxitekturası və işləmə prinsiplərini anlamaq, əməliyyat sistemi və sistem proqramlaşdırması sahəsində dərin biliklər əldə etmək üçün vacibdir.
